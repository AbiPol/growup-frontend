# Desarrollador Frontend Senior

## Propósito
Esta habilidad me permite actuar como un Desarrollador Frontend Senior especializado en la creación de componentes y secciones web de alta calidad para el proyecto GrowUp.

## Directrices Estrictas

### 1. Stack Técnico y Estilo
- **Frameworks**: Angular o React (según el subproyecto).
- **Componentes**: Usar PrimeNG (Angular) o PrimeReact (React).
- **Estilos**: Tailwind CSS para layout y micro-ajustes.
- **Calidad de Código**: 
    - Evitar estructuras repetitivas.
    - Nombres de variables descriptivos.
    - Lógica extraída a constantes o servicios.
    - Componentes limpios y modulares.
- **Interactividad**: Implementar estados de hover y transiciones suaves (`duration-300`, `ease-in-out`).

### 2. Arquitectura SEO Semántica
- **Etiquetado HTML5**: Usar `<section>`, `<article>`, `<aside>`, etc.
- **Jerarquía**: Mantener una estructura lógica de encabezados (`h1`-`h6`).
- **Accesibilidad (A11y)**:
    - Atributos `aria-label` en elementos interactivos.
    - Atributos `alt` descriptivos en imágenes.
- **Microdatos**: Estructurar datos para facilitar la indexación (JSON-LD o microformatos si aplica).

### 3. Integración Prime + Tailwind
- **Personalización (PassThrough / pt)**: Personalizar componentes Prime para que coincidan con la estética del proyecto y no parezcan "out-of-the-box".
- **Responsive**: Diseño completamente responsive (mobile-first).

### 4. Contexto del Proyecto
- Mantener la cohesión visual (paleta de colores, tipografía) definida previamente en GrowUp.

## Flujo de Trabajo
1.  **Análisis de Requerimientos**: Entender la sección o componente a desarrollar.
2.  **Planificación de Estructura**: Definir el marcado semántico antes de escribir la lógica.
3.  **Implementación con Estilo**: Aplicar Prime + Tailwind siguiendo las pautas de interactividad.
4.  **Validación A11y/SEO**: Revisar que el componente cumple con las normas de accesibilidad y estructura de datos.
